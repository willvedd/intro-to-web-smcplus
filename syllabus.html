Course Objectives

At the end of this course, students should understand basic principles of web design and development, including the knowledge and skills required to edit existing websites as well as create new ones.

Course Description

This course will introduce students to basic concepts and practices in web design and development. Students will learn how to effectively plan, prepare and structure a website based on current best-practices. Students will then learn to build a website with HTML and style their sites using CSS.

At the end of this course, students will be able to:

  • Plan the architecture of a website
  • Structure a website document with HTML
  • Adjust the visual styles of a web document using CSS
  • Gain an overview of the design and development process
  • Edit any existing website

This course is an asynchronous course. Students will watch video lectures on their own time and at their own pace. Each lecture will conclude with a series of exercises designed to test/implement the understanding/knowledge gained during the lecture. Students will then be required to participate in one-hour synchronous web meetings with the instructor. Culminating Project

This course will feature a culminating project due at the end of the five weeks. Students are to create a webpage. The subject matter of the page could be anything but students are encouraged to focus on something useful like a resume or portfolio.

Your final project must meet five of the following seven criteria to receive a passing grade:

  • At least seven different, correctly-used HTML tags - Be sure to use open and close all your tags correctly. Double check with an HTML validator
  • At least three of the colors used from your color palette
  • At least ten different CSS properties - Remember the simple ones: font-size, margin, padding, text-align, color, background-color etc. Comprehensive list of properties.
  • Your projected hosted on your github page
  • At least two external files - Files could be anything, images, videos, mp3s, pdfs, word documents. Integrated it into your project through any means necessary. So long as they're used in your project in some capacity.
  • At least one media query - It doesn't matter what property or class your media query affects. As long as it works.
  • An external stylesheet -

Final project is due via email (will@bluecadet.com) by Monday, February 13 at 8pm. Please zip all your project files into a single email attachment. Please email me with any questions.

Schedule

Week 1 - Introduction to the Web

Lecture

What makes a good website, how thoughtful planning can greatly assist the design and development process, looking into website's information architecture, the 10,000ft view of the design process, introduction to user-experience (UX), some modern design trends and caveats.

Exercise

User-experience message board response.

Meeting

Introduction to what HTML is and the tag syntax. Review of basic HTML tags and how they should be applied. Looking at global and specific HTML attributes that can extend functionality. Learn about valid markup, validation techniques and browser forgiveness.

Assigned Work

Complete small HTML challenge. Revise design or information architecture in accordance with feedback.

Week 2 - Continuing with HTML

Lecture

Continued practice with HTML, specifically introducing the topic of writing semantic HTML and the benefits that come along with it. Introduction into different tags. Introducing students with the web inspector and a more in-depth look into what the browsers actually does with your code.

Exercise

Message board response.

Meeting

Wrapping up discussion of a semantic HTML document. More advanced HTML elements, best practices and common gotchas.

Assigned Work

Create an external stylesheet and begin applying some styles. Focus more on basic; don't worry about layout and sizing just yet.

Week 3 - Introduction to Styling

Lecture

Introduction to styling through the "inline" style attribute and some common style properties. Abstracting styles from your HTML with cascading style sheets (CSS). Learning to write syntactically correct CSS, class and ID selectors and what the "C" in CSS means.

Meeting

Continued practice with CSS. Looking into two of the more powerful CSS properties: display and position. How we can use those two properties to create site layouts. Common techniques and gotchas.

Exercise

Message board response. Send me samples of your project progress.

Week 4 - Principles of Responsive Web Design

Lecture

Focusing on Responsive Web Design (RWD) principles, why it's important and why you should learn and adopt it. How to implement RWD into your web project by avoiding certain units, utilizing certain style properties and media queries.

Meeting

Introduction to pseudoelements and leveraging more advanced CSS properties like transition and animation to create interactive pages. Addressing the harsh realities of the browser: inconsistencies, prefixes and testing. Allowing for catch-up of HTML/CSS, addressing any questions and concerns.

Exercise

Message board response. Send me samples of your project progress.

Week 5 - Wrap-up and Other Important Takeaways

Lecture

Wrapping up anything and reviewing HTML/CSS. Going over some more general web development things like common front-end frameworks and content management systems (CMS), paid Web Services and advice on how to continue learning and creating beyond the course.

Meeting

The harsh realities of web design and development.

Exercise

Message board response. Send me samples of your project progress.

Grading

This course is graded on a pass/fail basis. In order to pass, students must:

  • Complete final project (requirements above)
  • Complete all five message board responses
  • Be prepared for and actively participate in all asynchronous meetings
